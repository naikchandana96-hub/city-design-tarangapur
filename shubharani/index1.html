üö¢ Port-Based Ship Repair & Maintenance Hub

Algorithm: Assembly-Line Scheduling (Dynamic Programming)

This dynamic programming approach minimizes total completion time by finding the fastest path through a series of multi-stage repair workstations (e.g., Hull, Engine, Paint), accounting for time taken to transfer between lines.

  • **Time Complexity:** $O(N)$ (where N is the number of stations)
  • **Use Case:** Scheduling multi-stage repair tasks across workstations to minimize total completion time.
// C++ Code for Assembly Line Scheduling (Dynamic Programming) #include <bits/stdc++.h> using namespace std; // Calculates minimum time to complete repair across 2 assembly lines int carAssembly(int a[][4], int t[][4], int e[], int x[]) { int n = 4; // 4 stations (N) int T1[4], T2[4]; // Entry time to first station T1[0] = e[0] + a[0][0]; T2[0] = e[1] + a[1][0]; // Fill tables using DP recurrence for (int i = 1; i < n; ++i) { T1[i] = min(T1[i - 1] + a[0][i], T2[i - 1] + t[1][i] + a[0][i]); T2[i] = min(T2[i - 1] + a[1][i], T1[i - 1] + t[0][i] + a[1][i]); } // Exit times return min(T1[n - 1] + x[0], T2[n - 1] + x[1]); } int main() { int a[2][4] = {{4, 5, 3, 2}, {2, 10, 1, 4}}; // Station processing times int t[2][4] = {{0, 7, 4, 5}, {0, 9, 2, 8}}; // Transfer times int e[2] = {10, 12}; // Entry times int x[2] = {18, 7}; // Exit times cout << "Minimum Repair Time: " << carAssembly(a, t, e, x) << " hours" << endl; return 0; }

üèñ Coastal Food Truck Fleet Management

Algorithm: Multiple Knapsack + Bin Packing (Dynamic Programming / Greedy)

This is a variation of the knapsack problem, where we determine the optimal set of inventory items (value/profit) to pack into multiple trucks (capacity/weight) going to different zones. It maximizes potential revenue based on space limits.

  • **Time Complexity:** $O(N \cdot W)$ (for DP on a single knapsack, extended for multiple)
  • **Use Case:** Optimizes food-truck routes and inventory distribution across coastal zones.
// C++ Code for 0/1 Knapsack (Core logic for inventory loading) #include <bits/stdc++.h> using namespace std; // Returns max value that can be put in knapsack of capacity W int knapSack(int W, int wt[], int val[], int n) { int i, w; // K[i][w] stores the maximum value with 'i' items and capacity 'w' vector<vector<int>> K(n + 1, vector<int>(W + 1)); for (i = 0; i <= n; i++) { for (w = 0; w <= W; w++) { if (i == 0 || w == 0) K[i][w] = 0; else if (wt[i - 1] <= w) // Take item or leave item (Dynamic Programming choice) K[i][w] = max(val[i - 1] + K[i - 1][w - wt[i - 1]], K[i - 1][w]); else K[i][w] = K[i - 1][w]; // Item is too heavy } } return K[n][W]; } int main() { int val[] = {60, 100, 120}; // Sales value of items (Profit) int wt[] = {10, 20, 30}; // Weight/Space of items (Capacity) int W = 50; // Single Truck Capacity int n = sizeof(val) / sizeof(val[0]); cout << "Max Inventory Value for Truck: " << knapSack(W, wt, val, n) << endl; return 0; }

üè¶ Customer Service & Public Service Queue Management

Algorithm: Round-Robin + Multi-Level Priority Queues

This hybrid system ensures fairness for the general public (Round-Robin) while immediately escalating and processing emergency cases (Priority Queue), preventing system collapse during critical demand peaks.

  • **Time Complexity:** $O(\log N)$ (for heap operations on the priority queues)
  • **Use Case:** Maintains fair queues while prioritizing emergency cases (e.g., bank, government office).
// C++ Code for Multi-Level Priority Queue (Simulating Emergency Dispatch) #include <bits/stdc++.h> using namespace std; // Pair stores {Priority, TicketID} void processQueues(priority_queue<pair<int, string>>& emergency, queue<string>& standard) { cout << "--- Processing Queue ---\n"; // 1. Process all high-priority cases first while (!emergency.empty()) { auto task = emergency.top(); emergency.pop(); cout << "EMERGENCY (P" << task.first << "): " << task.second << " handled.\n"; } // 2. Process standard queue (Round-Robin logic can be layered here) int count = 0; while (!standard.empty() && count < 3) { // Process 3 standard tasks before checking again cout << "Standard: " << standard.front() << " handled.\n"; standard.pop(); count++; } if(!standard.empty()) cout << "Remaining standard tasks in queue...\n"; } int main() { // Max-Heap for Emergency/High Priority (Higher number = Higher priority) priority_queue<pair<int, string>> emergency_q; // FIFO Queue for Standard Service queue<string> standard_q; // Add tasks standard_q.push("Ticket-A"); emergency_q.push({10, "Crisis-101"}); // High priority standard_q.push("Ticket-B"); emergency_q.push({5, "Urgent-202"}); // Medium priority standard_q.push("Ticket-C"); standard_q.push("Ticket-D"); processQueues(emergency_q, standard_q); return 0; }

üßò Mental Health & Rehabilitation Hub

Algorithm: K-Means Clustering

An unsupervised learning algorithm that groups patients into compatible therapy clusters based on shared attributes (e.g., condition, age, social preference), maximizing the effectiveness of group therapy sessions.

  • **Time Complexity:** $O(k \cdot I \cdot N \cdot D)$ ($k$ clusters, $I$ iterations, $N$ patients, $D$ dimensions)
  • **Use Case:** Groups patients into compatible therapy clusters.
// Python Conceptual Code for K-Means Clustering (Data Grouping) // (C++ equivalent is complex, Python is standard for ML/Clustering demonstration) # Sample Patient Data: [Age, Anxiety_Score, Social_Openness] data = [ [25, 8.5, 9.1], # A [27, 8.8, 9.0], # B [65, 3.2, 4.5], # C [68, 3.0, 4.2], # D [45, 6.0, 7.0], # E ] // K-Means Steps: // 1. Initialize K centroids (e.g., K=2: Young Group vs. Senior Group) // 2. For each patient, calculate distance to each centroid. // 3. Assign patient to the closest cluster. // 4. Recalculate the centroid (mean) of each new cluster. // 5. Repeat steps 2-4 until centroids stabilize. from sklearn.cluster import KMeans import numpy as np X = np.array(data) kmeans = KMeans(n_clusters=2, random_state=0, n_init=10).fit(X) print("Cluster Labels for Patients (0 or 1):", kmeans.labels_) print("Centroids (Mean Profile of each Cluster):\n", kmeans.cluster_centers_) // Patients A, B likely Cluster 0 (Young, High Score) // Patients C, D likely Cluster 1 (Senior, Low Score) // Patient E is mid-range, assigned to closest cluster
